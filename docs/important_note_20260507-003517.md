# 重要変更記録: FortniteReplayReader アップグレード & Unreal.Core パッチ適用

**日時**: 2026-05-07 00:35 JST  
**背景**: Fortnite が 2026-05-06 深夜（JST）にゲームアップデートを実施。新リプレイフォーマットに移行したことで、既存の `FortniteReplayReader 2.4.0 (net9.0)` が全リプレイ解析に失敗するようになった。

---

## 発生した問題

`replay_parser` サービスの `/api/replay-to-json` および `/api/replays/parse` が、2026-05-07 以降に保存されたリプレイに対して HTTP 422 を返し続ける状態になった。

**エラー内容**:
```
System.IndexOutOfRangeException: Index was outside the bounds of the array.
   at Unreal.Core.ReplayReader`1.ReadNetFieldExports(FArchive archive)
   at Unreal.Core.ReplayReader`1.ReadExportData(FArchive archive)
   at Unreal.Core.ReplayReader`1.ReadDemoFrameIntoPlaybackPackets(FArchive archive)
   at Unreal.Core.ReplayReader`1.ReadReplayData(FArchive archive, Int32 fallbackChunkSize)
   at Unreal.Core.ReplayReader`1.ReadReplayChunks(FArchive archive)
   at FortniteReplayReader.ReplayReader.ReadReplay(String fileName)
```

**根本原因**:  
`Unreal.Core` の `ReadNetFieldExports` メソッドが、NetFieldExport の `Handle` 値がグループの配列サイズを超えた際に配列の動的拡張を行わず、`IndexOutOfRangeException` をスローしていた。Fortnite の新パッチで既存の ExportGroup に新フィールドが追加され、Handle 値が事前確保済みの配列長を超えるようになったことで発症。

---

## 実施した変更

### 1. .NET ターゲットフレームワーク & NuGet パッケージのアップグレード

**ファイル**: `services/replay_parser/Fortnite_Replay_Parser_GUI.csproj`

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| `TargetFramework` | `net9.0` | `net10.0` |
| `FortniteReplayReader` バージョン | `2.4.0` | `3.0.0` |

`FortniteReplayReader 3.0.0` は `net10.0` 以上を要求するため、ターゲットフレームワークも同時にアップグレードした。  
.NET 10 SDK (10.0.203) は `dotnet-install.ps1` を使用してユーザーローカルパス (`%LOCALAPPDATA%\Microsoft\dotnet`) にインストール済み。アップグレード後、PATH 上の `dotnet.exe` も 10.0.203 に更新されている。

### 2. Unreal.Core ライブラリへのパッチ適用

`FortniteReplayReader 3.0.0` にアップグレードしても同一クラッシュが継続したため、依存ライブラリ `Unreal.Core 3.0.0` のソースを取得してパッチを適用した。

**ソースリポジトリ**: `https://github.com/Shiqan/FortniteReplayDecompressor`  
**パッチ対象ファイル**: `src/Unreal.Core/ReplayReader.cs` の `ReadNetFieldExports` メソッド

**変更箇所** (`ReadNetFieldExports` 内、`ReadNetFieldExport` 結果を配列に書き込む直前):

```csharp
// ---- 変更前 ----
var netField = ReadNetFieldExport(archive);
if (group != null && netField != null)
{
    group.NetFieldExports[netField.Handle] = netField;  // Handle が配列長超過で IndexOutOfRangeException
}

// ---- 変更後 ----
var netField = ReadNetFieldExport(archive);
if (group != null && netField != null)
{
    if (netField.Handle >= group.NetFieldExports.Length)
    {
        // Handle が配列長を超えている場合は動的拡張する
        var oldExports = group.NetFieldExports;
        group.NetFieldExports = new NetFieldExport[netField.Handle + 1];
        group.NetFieldExportsLength = netField.Handle + 1;
        Array.Copy(oldExports, group.NetFieldExports, oldExports.Length);
    }
    group.NetFieldExports[netField.Handle] = netField;
}
```

**注意**: 同じ `ReplayReader.cs` 内の別メソッド (`ReadNetFieldExports` の別オーバーロード、約 line 909) には既に `group.IsValidIndex(netField.Handle)` チェックが存在していた。今回修正したのは旧実装側（チェックのなかった箇所）のみ。

**適用方法**:  
ソースを `%TEMP%\FortniteReplayDecompressor` にクローン → パッチ適用 → `dotnet build -c Release` でビルド → 生成した `Unreal.Core.dll` を NuGet グローバルキャッシュ内の既存 DLL に上書き。

```
パッチ済み DLL の配置先:
C:\Users\<user>\.nuget\packages\unreal.core\3.0.0\lib\net10.0\Unreal.Core.dll
```

`replay_parser` サービスは `dotnet run` でビルドするため、次回ビルド時にパッチ済み DLL が自動的に使用される。

### 3. デバッグ用一時変更の適用と復元

診断のため `Program.cs` の `/api/replay-to-json` エラーレスポンスに `stackTrace` フィールドを一時追加し、原因特定後に復元した。最終的に `Program.cs` へのコード変更は残っていない（net変更なし）。

また `FortniteReplayHelper.cs` の `ParseMode.Full` を `ParseMode.Minimal` に一時変更したが、パッチ適用後に `ParseMode.Full` へ戻している。

## 今後の注意事項

1. **Unreal.Core の NuGet 更新時は再パッチが必要**  
   `FortniteReplayReader` や `Unreal.Core` のバージョンを上げた場合、NuGet キャッシュ内の DLL がオリジナルに戻るため、同じパッチを再適用する必要がある。  
   または、upstream が正式にこの修正を取り込んだ新バージョンをリリースした際に、そちらへ移行するのが望ましい。

2. **パッチ済みソースの保存場所**  
   パッチを当てたソースは一時ディレクトリ (`%TEMP%\FortniteReplayDecompressor`) にある。消えた場合は再クローン + 同一パッチを適用すること。

3. **ParseMode.Full は維持**  
   `FortniteReplayHelper.cs` は `ParseMode.Full` を使用している。`ParseMode.Minimal` でも動作するが、Full でなければ取得できないフィールドが今後使われる可能性があるため変更しない。
